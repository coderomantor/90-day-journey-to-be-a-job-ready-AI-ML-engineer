## Commit & Repository Quality Rules

Whenever I say "commit", "prepare commit", "push", or ask you to finalize work, you must follow this checklist before suggesting or making a commit.

### 1. Review Changed Files

- Run `git status`.
- Identify modified, created, deleted, and untracked files.
- Do not commit random scratch files, temporary files, cache files, test files, or unrelated files.
- If a file looks unrelated, ask me before including it.

### 2. Check Professional Quality

Before committing, verify that:

- Folder structure matches the existing repository style.
- Documentation follows the style of previous days.
- README files are clear, professional, and beginner-friendly.
- Code is readable, simple, and not over-engineered.
- File names are clean and professional.
- Screenshots/images have meaningful names.

### 3. Check Metadata and Unwanted Files

Do not commit:

- `.env`
- API keys
- passwords
- tokens
- private credentials
- `__pycache__/`
- `.pytest_cache/`
- `.DS_Store`
- random notebook checkpoints
- temporary test files
- unnecessary system-generated files

If any of these appear, stop and warn me.

### 4. Run Basic Validation

Before commit, check where applicable:

- Python files can run without obvious syntax errors.
- Requirements files are updated if new packages are used.
- Links in README are not obviously broken.
- Project folder contains the expected files.
- Screenshots/output files are in the correct folder.

### 5. Commit Message Rules

Use clear, professional commit messages.

Preferred format:

- `Add Day 04 data visualization project`
- `Update roadmap progress for Day 04`
- `Improve Day 03 documentation`
- `Clean repository structure`

Avoid vague messages like:

- `update`
- `changes`
- `final`
- `fixed stuff`

### 6. Ask Before Pushing

Unless I explicitly say "push", do not push automatically.
First show me:

- Files to be committed
- Recommended commit message
- Any warnings
- Final checklist result

### 7. Final Response Format

Before committing, respond with:

Commit Review:

- Files included:
- Files excluded:
- Quality check:
- Metadata/secrets check:
- Suggested commit message:
- Safe to commit? Yes/No

Only proceed when the repo is clean, intentional, and professional.
